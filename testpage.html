Standard mailto href — display text matches href

Mailto href with a different display label

Contact Billing

Plus-addressing, subdomain sender, dots in local part

alerts@mail.notifications.bigcorp.com

first.last.middle@domain.co.uk

Expected: all three replaced normally

5. Non-Email @ Symbols (Should Not Change)

@ mentions and code snippets

Expected: neither @support nor the shell snippet is treated as an email address

6. Email in an Input Field

Input with email as placeholder and value

Expected: both the value= and placeholder= attributes are replaced

7. Email in a Textarea

Textarea with prefilled email content

Expected: both email addresses in the textarea body are replaced

8. Email in a Select Dropdown

Select with email addresses as option values and labels

value: alice@team.com

Expected: visible option labels are replaced with masked addresses; the value= attribute is not replaced — confirm by selecting an option and checking the value shown to the right of the dropdown
