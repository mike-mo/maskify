Maskify Test Page

Click the Maskify button in the extension popup, then verify each scenario below matches its expected result.

1. Plain Text (no link)

Single email in a sentence

For questions, contact support@acme.com directly.

Expected: email replaced with a masked address like alice23***@example.com
Multiple different emails in one paragraph

Reach Alice at alice@company.com or Bob at bob@company.com, or copy both via team@company.com.

Expected: three different masked addresses — all distinct from one another

2. Mailto Links

Standard mailto href — display text matches href

info@example.org

Expected: both the visible text and the href= value are replaced with the same masked address
Mailto href with a different display label

Contact Billing

Expected: href= value is replaced; visible text "Contact Billing" is unchanged

3. Email Format Variations

Plus-addressing, subdomain sender, dots in local part

user.name+filter@newsletter.io

alerts@mail.notifications.bigcorp.com

first.last.middle@domain.co.uk

Expected: all three replaced normally

4. Tabular Data and Consistent Mapping

Emails inside a table — including a person listed twice
NameEmailDepartment
Carolcarol@hr.example.orgHR
Davedave.smith@ops.example.orgEngineering
Carolcarol@hr.example.orgIT Liaison
Expected: both Carol rows show the same masked address; Dave gets a different one

5. Non-Email @ Symbols (Should Not Change)

Social media handles and code snippets

Follow us @maskify on Twitter. In bash: echo $USER@$(hostname)

Expected: neither @maskify nor the shell snippet is treated as an email address

6. Email in an Input Field

Input with email as placeholder and value

Expected: both the value= and placeholder= attributes are replaced

7. Email in a Textarea

Textarea with prefilled email content

Expected: both email addresses in the textarea body are replaced

8. Email in a Select Dropdown

Select with email addresses as option values and labels

value: alice@team.com

Expected: visible option labels are replaced with masked addresses; the value= attribute is not replaced — confirm by selecting an option and checking the value shown to the right of the dropdown
